# Авангард — Vue 3

Проект на Vue 3 + Vite + Vue Router. Дизайн полностью соответствует исходному
HTML-макету — ничего визуально не менялось, только перенесено в компоненты.

## Запуск

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # сборка в dist/
npm run preview  # предпросмотр сборки
```

## Структура

```
src/
├── assets/styles/main.css   # все CSS из исходного макета (1:1)
├── data/                    # ВСЕ ТЕКСТЫ И ДАННЫЕ — менять здесь
│   ├── site.js              # компания, контакты, навигация
│   ├── home.js              # главная
│   ├── materials.js         # материалы
│   ├── equipment.js         # спецтехника
│   └── services.js          # услуги
├── components/
│   ├── layout/              # AppNav, AppFooter — общие для всего сайта
│   ├── ui/                  # «кирпичики» (StatCard, CatalogCard, …)
│   └── sections/            # секции страниц (PageHero, WhyUsSection, …)
├── composables/
│   └── useScrolled.js       # отслеживание прокрутки для шапки
├── views/                   # страницы (по одной на маршрут)
├── router/                  # Vue Router + scrollBehavior для якорей
├── App.vue
└── main.js
```

## Архитектура: как и где править

### Поменять контакты, телефон, режим работы
→ `src/data/site.js`. Меняется одновременно везде — и в шапке, и во всех
секциях контактов на 4 страницах.

### Поменять цены или добавить позицию в каталог
→ `src/data/materials.js` (или `equipment.js`, `services.js`), массив
`catalog.items`. Карточка отрисуется автоматически, ничего больше менять
не нужно.

### Поменять текст блока «Почему выбирают нас»
→ соответствующий файл в `data/`, объект `whyUs`. Иконки, заголовки,
описания — всё там.

### Добавить новую страницу
1. Создай `src/data/newpage.js` с данными.
2. Создай `src/views/NewPageView.vue`, импортируя нужные секции.
3. Добавь маршрут в `src/router/index.js`.
4. При желании добавь ссылку в `src/data/site.js → navLinks`.

### Поменять дизайн (цвета, отступы, шрифт)
→ `src/assets/styles/main.css`. Корневые переменные в `:root`:
`--bg`, `--fg`, `--mid`, `--border`, `--card`, `--radius`, `--max`, `--nav-h`.

## Принципы повторного использования

Одинаковые блоки (карточки каталога, «почему выбирают нас», контакты,
формы) — это **один компонент с props**. Контент задаётся через данные,
а не через копирование разметки.

Например, секция контактов на всех 4 страницах — это один и тот же
`ContactSection.vue`. Её отличия (заголовок, текст формы, показывать ли
адрес, режим работы) задаются в JSON-объекте `contact` в файле данных
страницы.

То же самое — каталог: `CatalogSection.vue` принимает массив `items`, и
не важно, материалы это, техника или услуги. Если у позиции нет цены
(как у услуг) — поле просто не передаётся, и блок цены не отрисуется.

## Форма заявки

`ContactForm.vue` использует Vue-реактивность: валидация и состояние
«отправлено» — без `getElementById`, как в исходнике. Чтобы подключить
реальную отправку на бэк — найди в `ContactForm.vue` функцию `submit()`
и добавь fetch-вызов перед `submitted.value = true`.
